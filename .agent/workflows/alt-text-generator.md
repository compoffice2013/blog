このワークフローは、ブログ記事に添える画像のALTテキスト（代替テキスト）とキャプションを自動生成します。
Rank Math SEO 100点対策の画像ALT属性要件を満たすために使います。

## 手順

1. **ペルソナの確認**
   - `00_persona/persona.md` でルール・禁止語を確認
   - 特にNGワード（「魔法」「特別な」「体験」）を避ける

2. **画像の分析**
   - ユーザーから提供された画像を確認する
   - 画像に写っている内容を詳細に把握する

3. **ALTテキストの生成**
   - **要件**:
     - フォーカスキーワードを含める（SEO対策）
     - 画像の内容を簡潔かつ具体的に説明する
     - 50〜100文字程度
     - NGワードを使用しない
   - **例**: 「富山市の料亭でマジックショーを披露するマジシャン・コンプレッサーとお客様の笑顔」

4. **キャプションの生成**
   - **要件**:
     - 読者の興味を引くひと言
     - 記事の文脈に合った表現
     - 30〜50文字程度
   - **例**: 「笑顔あふれる新年会のひととき」

5. **出力形式**
   - コピーしやすい形式で提示：

   ```
   【ALTテキスト】
   富山市の料亭でマジックショーを披露するマジシャン・コンプレッサーとお客様の笑顔

   【キャプション】
   笑顔あふれる新年会のひととき
   ```

## 複数画像の場合

画像が複数ある場合は、以下の形式で番号付きで出力する：

```
【画像1】
ALT: ○○○○
キャプション: ○○○○

【画像2】
ALT: ○○○○
キャプション: ○○○○
```

## SEO関連の参考

- [[04_knowledge/rank_math_seo_guidelines|Rank Math SEO 100点獲得ガイドライン]]
- 100%スコアには4つ以上の画像/動画が必要
- すべてのALT属性にフォーカスキーワードを含める

## 注意事項

- 常に日本語で出力してください。
- 画像の生成は行いません（`persona.md` のルール「明示的な指示があるまで勝手に生成しない」を遵守）。
