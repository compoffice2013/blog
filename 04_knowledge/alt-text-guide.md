# 最適なALTテキストの書き方：2026年版 最新ガイド

## はじめに

ALT（代替）テキストは、ウェブ上の画像が持つ情報をテキストで説明するものです。視覚に障害を持つユーザーがスクリーンリーダーでコンテンツを理解するため、また検索エンジンが画像を正しく解釈するために不可欠な要素となっています。近年、ウェブアクセシビリティへの要求の高まりや、AIを活用した画像検索技術の進化に伴い、ALTテキストの重要性はかつてないほど高まっています。

欧州連合では2025年6月から欧州アクセシビリティ法（EAA）が施行され、ALTテキストの不備は法令違反と見なされる可能性があります [2]。また、SEOの観点からも、適切に設定されたALTテキストは画像検索からのトラフィックを大幅に増加させることが報告されています [2]。

本ガイドでは、W3Cの公式ガイドラインを基盤とし、2026年に向けての最新のベストプラクティス、AI時代におけるトレンド、そして具体的な実装方法までを包括的に解説します。

## 1. ALTテキストの基本：何を、いつ、どのように書くか

ALTテキストを適切に設定するための最初のステップは、その画像が持つ「目的」を理解することです。W3Cは、画像の目的を判断するためのデシジョンツリー（決定木）を提供しており、これに従うことで、どのようなALTテキストを記述すべきか、あるいは不要かを判断できます [1]。

### W3C ALTデシジョンツリーの要約

| 画像の状況 | 推奨される対応 |
| :--- | :--- |
| **テキストを含む画像** | テキストが画像以外に存在しない場合、そのテキストをALTに記述する。装飾目的のテキストならALTは空にする。 |
| **リンクやボタンとして機能する画像** | 画像がなければ機能が理解できない場合、リンク先や実行されるアクションをALTに記述する。（例：「プロフィールページへ」） |
| **意味を持つ情報的な画像** | 画像が伝える情報を簡潔に説明する。（例：「グラフや図」の場合は、画像の内容を説明するテキストを別途用意する） |
| **純粋に装飾的な画像** | ページのデザイン目的で使われ、情報を伝えない画像。ALT属性を空（`alt=""`）にする。 |

重要なのは、**ALT属性を省略するのではなく、装飾画像の場合は必ず空の`alt=""`を設定する**ことです。属性自体を省略すると、スクリーンリーダーがファイル名を読み上げてしまい、ユーザー体験を損なう原因となります [2]。

## 2. 2026年を見据えたALTテキストのベストプラクティス

基本的な考え方を理解した上で、より効果的なALTテキストを作成するための最新のベストプラクティスを紹介します。

### ALTテキスト作成の黄金ルール

1.  **説明的かつ簡潔に**: 画像が何を伝えているかを具体的に記述しつつ、スクリーンリーダーの読み上げに配慮して125文字以下に収めるのが理想です [2]。
2.  **「〜の画像」は不要**: スクリーンリーダーはそれが画像であることを自動で認識するため、「〜の画像」や「〜の写真」といった記述は冗長になります [5]。
3.  **コンテキストを意識する**: なぜその画像がその場所にあるのかを考え、ページの文脈に沿った説明を加えることで、より価値のある情報を提供できます [2]。
4.  **キーワードの詰め込みは避ける**: SEOを意識するあまり、不自然にキーワードを羅列するのはやめましょう。あくまでもユーザーにとって自然で分かりやすい説明を心がけることが、結果的にSEOにも良い影響を与えます [3]。

### 具体例で見る良いALTテキストと悪いALTテキスト

| 画像の種類 | 悪い例 | 良い例 |
| :--- | :--- | :--- |
| **商品画像** | `alt="シャツ"` | `alt="メンズ ブルー コットンTシャツ、クルーネック"` |
| **情報的な写真** | `alt="会議"` | `alt="ホワイトボードでシステムアーキテクチャ図について議論する3人の開発者"` |
| **機能を持つロゴ** | `alt="ロゴ"` | `alt="[会社名] - ホームページに戻る"` |

## 3. 画像の種類別 詳細ガイド

### 複雑な画像：グラフ、チャート、インフォグラフィック

グラフやインフォグラフィックのように多くの情報を含む複雑な画像には、単一のALTテキストだけでは不十分です。W3Cは、このような場合に**二部構成の代替テキスト**を推奨しています [4]。

1.  **短い説明（`alt`属性）**: 画像が何であるかを簡潔に示し、詳細な説明がどこにあるかを示唆します。（例: `alt="2024年売上推移グラフ、詳細は下記参照"`）
2.  **長い説明（Long Description）**: 画像が示すデータの詳細、傾向、結論などをテキストで提供します。

長い説明を提供するには、以下のような方法があります。

*   **画像に隣接するテキストリンク**: 長い説明を記載した別ページやページ内アンカーへのリンクを設置します。
*   **HTML5の`<figure>`と`<figcaption>`**: 画像と、その説明文（キャプション）を意味的にグループ化します。これは現在最も推奨される方法の一つです。

```html
<figure role="group">
  <img src="chart.png" alt="2024年売上推移グラフ、詳細は下記参照">
  <figcaption>
    <h3>2024年売上推移</h3>
    <p>このグラフは、2024年における四半期ごとの売上推移を示しています。第1四半期には1,000万円だった売上が、第4四半期には1,500万円に増加し、年間を通じて安定した成長が見られました。</p>
    <!-- より詳細なデータテーブルをここに含めることも可能 -->
  </figcaption>
</figure>
```

### AIとALTテキストの未来

AI技術の進化は、ALTテキストの生成と活用にも大きな影響を与えています。AIによる画像認識は、より正確で文脈に沿ったALTテキストの自動生成を可能にしつつあります [3]。特に、AIを搭載したビジュアル検索エンジンは、ALTテキストを重要なランキング要因として利用しており、その最適化は今後さらに重要になるでしょう。

## 4. よくある間違いとテスト方法

### よくある6つの間違い

| 間違い | 解決策 |
| :--- | :--- |
| 1. `alt`属性自体を記述しない | 装飾的な画像であっても必ず`alt=""`を記述する。 |
| 2. 「〜の画像」と記述する | スクリーンリーダーが自動で読み上げるため不要。 |
| 3. 説明が長すぎる | 125文字を目安に、最も重要な情報から記述する。 |
| 4. 重要な情報を後に書く | ユーザーがすぐに理解できるよう、結論や要点を先に書く。 |
| 5. 宣伝文句を入れる | 画像の内容のみを客観的に説明する。 |
| 6. 説明が必要か迷って省略する | 迷った場合は、説明を記述しておく方が安全。 |

### ALTテキストのテスト方法

作成したALTテキストが適切に機能するかを確認するには、実際にスクリーンリーダーでテストすることが最も効果的です。

*   **主要なスクリーンリーダー**: JAWS (Windows, 有料), NVDA (Windows, 無料), VoiceOver (macOS/iOS, 標準搭載), TalkBack (Android, 標準搭載)
*   **ブラウザ拡張機能**: `Alt & Meta viewer` (Chrome) などのツールを使えば、ページ内のALTテキストを一覧で簡単に確認できます。
*   **オンラインチェッカー**: `WAVE`や`SEOptimer`などのウェブサイト評価ツールも、ALTテキストのチェック機能を提供しています。

## 5. 主要プラットフォームでの実装

| プラットフォーム | 実装方法 |
| :--- | :--- |
| **HTML** | `<img>`タグに`alt`属性を直接記述します。 |
| **WordPress** | メディアライブラリや投稿編集画面の「代替テキスト」フィールドに入力します。 |
| **Shopify** | 商品管理画面のメディア編集セクションで「代替テキストを編集」から入力します。 |

## まとめ

最適なALTテキストは、ウェブサイトをすべての人にとってアクセスしやすく、検索エンジンにとって理解しやすいものにするための鍵です。本ガイドで紹介したW3Cの原則と最新のベストプラクティスを実践することで、ユーザー体験を向上させ、SEO効果を最大化し、そして法的なリスクを低減させることができます。AI時代においても、その本質的な重要性は変わることはありません。常にユーザーの視点に立ち、画像の意図を的確に伝えることを心がけましょう。
